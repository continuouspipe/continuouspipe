Note: In order to open a bash session in this container, you need to have cp-remote installed and configured.

cp-remote bash -i -e {{environment.identifier}} -s {{component.name}} -f {{environment.flow.uuid}}

If /bin/bash is not available on the pod, you can specify the command you want to run:

cp-remote exec -i -e {{environment.identifier}} -s {{component.name}} -f {{environment.flow.uuid}} -- /bin/sh
